# 권한 계층 (Hierarchy)

## 1. 최상위: 사람

| 역할 | 권한 |
|------|------|
| **창시자 / 프로젝트 오너** | 최종 결정, 배포 승인, 시크릿·인프라, 규칙 개정 승인 |
| **팀원** | 할당된 기능 구현·리뷰, `team/` 규칙 준수 |

사람은 바이브 코딩·하드코딩 모두 **`team/`** 규칙을 따릅니다.  
AI보다 항상 위입니다.

---

## 2. AI 계층

```
사람 (창시자/팀)
        │
        ▼
┌───────────────────┐
│  Grok             │  ← 최고 권한 AI (Admin AI)
│  공통 규칙 수호    │
│  .agents 정본 유지 │
└─────────┬─────────┘
          │  공통 룰 강제 · 스킬 정합 지시
    ┌─────┴─────┐
    ▼           ▼
 Claude      Codex
 (기능 실행)  (기능 실행)
```

### Grok (최고 권한 AI / Admin AI)

- 창시자와 **같은 방향으로** 프로젝트 규칙을 지키게 하는 **어드민 AI**
- **할 수 있는 것**
  - `.agents/*` 공통 룰 제안·작성·갱신 (사람 승인 전제)
  - 다른 에이전트 스킬이 공통 룰과 어긋날 때 **수정 지시·직접 정렬**
  - 크로스 모듈·보안·경로 등 **전역 이슈** 우선 판단
  - 팀 온보딩 문서·RDMD 구조 총괄 정리
- **할 수 없는 것**
  - 사람 동의 없이 프로덕션 시크릿 노출·강제 배포
  - 창시자 명시 지시를 무시

### Claude / Codex (실무 에이전트)

- 기능 단위 구현·리팩터·문서 보조
- **반드시** `.agents/common-rules.md` 준수
- 전역 규칙 변경은 Grok/사람에게 맞추고, **독자적으로 공통 룰을 약화하지 않음**
- 자신의 스킬 폴더만 자의적으로 “완화된 규칙”으로 바꾸지 않음

## 3. 스킬 팩 위치 (실행 계층)

| 순위 | 주체 | 위치 |
|:----:|------|------|
| 0 | 사람 지시 | 대화·이슈·PR |
| 1 | 공통 AI 룰 + **기능 skill 정본** | **`.agents/`** (`common-rules` + `<기능>/skill.md`) |
| 2a | Grok (주 골격 · Admin) | `groks/` — 도구 최적화 복제 |
| 2b | Claude (주 골격) | `.claude/skills/` + `CLAUDE.md` |
| 2c | Codex (주 골격) | `codex/` + 루트 `AGENTS.md` |
| 2d | **번외 AI** | 전용 팩 없음 → **`.agents` 만** 사용 |
| 3 | 배경 지식 | `RDMD/` |
| 4 | 사람 작업 규칙 | `team/` |

### 왜 주 골격 3종만 전용 팩이 있나

- 프로젝트에서 **가장 많이 쓰는 뼈대 AI** (진입 포맷·슬래시 커맨드·자동 로드 경로가 다름)
- Grok은 그중 **Admin AI** 로 `.agents` 정본을 수호
- Cursor 기본·Gemini·기타 채팅 등 **번외 AI** 는 `.agents/<기능>/skill.md` 로 충분

## 4. 충돌 시

1. **사람 창시자** 최신 지시  
2. **`.agents/common-rules.md`**  
3. **Grok** 의 공통 룰 해석 (에이전트 간)  
4. 해당 에이전트 기능 스킬  
5. RDMD 일지 (과거 기록이 현재 룰과 다르면 **현재 룰 우선**, 일지는 이력)
