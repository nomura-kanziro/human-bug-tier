# 댓글 관리/상세 페이지 기능 개선 내역

## 주요 변경점 (커밋: 36c49d1)

### 1. 상세 페이지 진입/이동 및 404 오류 해결
- URL 파싱 로직 개선: `.html` 확장자 유무와 상관없이 id 파라미터를 정확히 추출하도록 정규식 적용
- 잘못된 접근 시(파라미터 없음) 안내 메시지 및 목록 복귀 버튼 표시
- 상세/목록 이동 시 `.html` 확장자 없는 경로도 지원

### 2. 댓글 관리 테이블 및 상세 UI/UX 강화
- 답변 필터(전체/신고/사유/검색) 및 토글 기능 개선
- 신고사유(⚠️) 팝업 UI 통일, 위치/닫힘 동작 보완
- 전체/신고 답변 필터 버튼, 검색, 셀렉트 등 UI/UX 강화
- 목록/상세 삭제, 차단, 페이지네이션 등 관리 기능 유지
- 코드 정리 및 중복 함수 제거

### 3. 삭제/차단 기능 개선
- 댓글 삭제 시 인덱스 기반에서 id 기반으로 변경 (데이터 무결성 강화)
- 전체 삭제/차단 버튼 이벤트 등록을 DOMContentLoaded 이후로 이동해 안정성 향상
- 차단 추가 시 중복 방지 및 입력값 초기화

### 4. 기타
- 상세/목록 이동 함수(goToDetail 등)에서 `.html` 확장자 없는 경로도 지원
- window.onload 대신 DOMContentLoaded 사용으로 렌더링 타이밍 일관성 확보

## 작업 목적
- 관리자 댓글 관리/상세 페이지의 UX 및 안정성 강화
- URL 구조 유연화로 404 오류 및 잘못된 접근 방지
- 관리 기능(삭제, 차단, 필터, 신고 등) 일관성 및 유지보수성 향상
