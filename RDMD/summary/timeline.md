# 개발 타임라인

RDMD 번호 체계를 기준으로 한 **대략적 개발 순서**입니다.  
같은 시기의 `frontend/*-record.md` ↔ `backend/*-record.md` (legacy 번호가 맞으면) 동일 작업 묶음으로 보면 됩니다.

```
[1~5]   공통 Header/Footer, getBasePath, 티어 페이지, CSS 정리
   │
[0~5]   Express + Mongoose 세팅, TierList CRUD  (backend)
   │
[10]    커스텀 메이커 골격 · Notice · TierList 신고/좋아요 확장
   │
[11~14] 공지 카테고리, 알림, 관리자 신고 컨트롤러
   │
[15~20] 게시판 · post_detail · 댓글 CRUD · 좋아요 · 신고
   │
[23]    로그인 · 회원가입 프론트 연동
   │
[24]    관리자 JWT 로그인 + 티어 신고 UI
   │
[25]    비밀번호 재설정 (랜덤 토큰 + SHA256)
   │
[26]    admin_api.js · 관리자 커스텀 표시 수정
   │
[27]    배포 통합 (getBasePath, render.yaml, 정적 서빙)
   │
[28]    requireAdmin 미들웨어 전면 적용
   │
[29]    관리자 댓글/공지 UI·인증 헤더 통일
   ▼
 현재 (풀스택 로컬 :5000 + Render 배포 가능)
```

## 번호 대역 가이드

| 대역 | 주제 |
|------|------|
| 1 ~ 10 | 초기 구조 · 공통 인프라 |
| 10 ~ 14 | 공지 · 알림 · 관리 신고 API |
| 15 ~ 22 | 커스텀 메이커 · 게시판 심화 |
| 23 ~ 26 | 인증 · 관리자 연동 |
| 27 ~ 29 | 배포 · 보안 · 관리 UX |

> **참고**: 15~24 구간은 초기 작업이 1~14에 집중되어 있어 번호 밀도가 낮고, 25번부터 주제 단위로 재정리된 기록이 많습니다. (`RDMD/README.md` 참고)
