# 04 — 금지사항 (Do Not)

깨면 **머지 보류** 수준입니다.  
AI가 제안해도 **사람이 거절**하세요.

## 보안 · 비밀

| 금지 | 이유 |
|------|------|

## 프로세스

| 금지 | 이유 |
|------|------|
| **`RDMD/commit_history/{본인}.md` 미작성 상태로 커밋·푸시** | 이력 공백 · 팀 규칙 위반 |
| **커밋 메시지 `:` 뒤 설명을 영어 문장으로만 작성** | 팀 이력 가독성 · 규칙 위반 |
| 팀원에게 프로덕션 시크릿 메신저 평문 전송 | 유출 |

| `.agents` / `team` 규칙 무시하고 “일단 동작만” 머지 | 기술 부채 |

### 커밋 메시지 형식 (필수)

형태: **`type(scope): 한국어 설명`**

| 구간 | 언어 | 예 |
|------|------|-----|
| **type** (및 선택 scope) | **영어** | `feat`, `fix`, `docs`, `chore`, `style`, `refactor` · `feat(admin)` |
| **`:` 뒤 한 줄 설명** | **한국어** | `공지 관리 페이지에 수정 기능 추가` |

- `git commit -m` 과 `commit_history` 의 **message** 는 위 형식을 따른다.  
- **금지**: 콜론 뒤까지 영어 문장 (`feat(admin): add notice edit`)  
- **금지**: type까지 한국어로 바꾸기 (`기능(관리자): ...`) — type은 영문 conventional 유지  
- 코드 식별자·파일 경로·API 이름 등 **고유 명사는 설명 안에 영어 그대로** 둘 수 있다.  
- AI가 영어 설명만 제안하면 **콜론 뒤만 한국어로 바꿔** 커밋한다.

예:
- ✅ `feat(admin): 공지 관리 페이지에 수정 기능 추가`
- ✅ `fix(auth): 계정 찾기 시 메일 미설정·발송 실패를 명확히 알림`
- ✅ `docs(commit_history): 커밋 해시 기입`
- ❌ `feat(admin): add notice edit on management page`
- ❌ `문서(규칙): 커밋 메시지 규칙 변경`
