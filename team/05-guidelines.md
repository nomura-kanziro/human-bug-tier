# 05 — 가이드라인 (권장)

금지(04)는 **빨간불**, 여기는 **노란불·권장**입니다.

---

## 바이브 코딩 vs 정석

| 방식 | 언제 | 반드시 지킬 것 |
|------|------|----------------|
| **바이브** (AI 많이 사용) | 프로토타입, UI 시안, 단순 CRUD | 01-rules, 04-prohibitions, 06-checklist, `:5000` 스모크, **commit_history** |
| **정석** | 인증·결제성 로직, 관리 권한, 배포 | + RDMD guides, 테스트 시나리오 문서화, 리뷰 |

바이브로 짜도 **금지사항은 동일**합니다.  
“AI가 알아서 했음”은 면책이 아닙니다.

## 리뷰 시 보면 좋은 것

1. 권한 헤더·미들웨어  
2. path/API base  
3. 에러/빈 상태 UX  
4. 시크릿·하드코딩 URL  
5. 변경 범위가 이슈와 일치하는지  

## 커밋 · 브랜치

- 한 커밋 = 한 의도  
- `feat` / `fix` / `docs` / `chore` 접두  
- `main`/`master` 직푸시 자제 (팀 운영 시 PR)  
- **필수**: 커밋·푸시 전 `RDMD/commit_history/{본인}.md` 작성  
  → [RDMD/commit_history/README.md](../RDMD/commit_history/README.md)

## 문서

- **매 커밋**: `commit_history` 개인 로그 (필수)  
- 기능 추가·보안·배포: `frontend`/`backend` record 권장  
- 규칙 변경: `team/` + `.agents/` 같이  

## AI 역할 가이드 (사람 입장)

| AI | 맡기면 좋은 일 |
|----|----------------|
| **Grok** | 전역 규칙, 경로/보안 정렬, RDMD·스킬 구조, 여러 에이전트 결과 통합 |
| **Claude** | 기능 단위 구현, 리팩터, 인수인계 문서 |
| **Codex** | 패치·테스트·반복 구현, AGENTS.md 기반 작업 |

공통: 시크릿 넣지 않기, 범위 명시하기, 결과는 team 체크로 검증.

## 커뮤니케이션

- 버그는 **재현 절차**와 함께  
- “안 됨” 대신 “어느 URL, 로그인 여부, 기대/실제”  
- 배포/장애는 창시자 또는 인프라 담당에게 즉시
